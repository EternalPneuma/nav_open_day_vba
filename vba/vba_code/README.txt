净值流程与工具使用说明
========================

本文档说明当前 VBA 工程中的 6 个主流程、3 个工具按钮、运行顺序、文件依赖、工作表依赖和注意事项。

建议阅读对象：日常操作人员、维护人员、后续接手该 Excel/VBA 流程的人员。


一、全部依赖总览
----------------

本流程默认所有文件都围绕当前主 .xlsm 所在目录运行。正式执行前，建议先按本节逐项检查。

1. 主工作簿

    - 当前文件必须为启用宏的 .xlsm。
    - 必须启用宏。
    - 必须包含 VBA 模块、用户窗体 frmDispatchConsole，以及入口宏 打开控制台。

2. 当前工作簿内工作表

    - Sheet1：产品维度表。
    - Sheet2：净值数据主表。
    - Sheet3：数据报告表，由 STEP2 创建或清空重建。

    Sheet1 字段要求：

        A 列：产品编号
        B 列：产品名称
        C 列：产品简称

    Sheet2 字段要求：

        A:J 为净值数据主表区域。
        A 列为净值日期。
        B 列为产品编号。
        D 列为净值。
        G 列为 30 日年化收益率。

3. 净值源数据表

    净值源数据表必须放在主 .xlsm 同目录，文件名格式必须为：

        净值数据浏览表 yyyy-mm-dd至yyyy-mm-dd.xlsx

    STEP1 会扫描同目录下所有符合该格式的文件，并按文件名中的开始日期升序处理。

    最新 Step1 已改为自动识别源数据起始行：

        - 优先在源文件 Sheet1 的 A 列查找“净值日期”，找到后从下一行开始导入。
        - 如果找不到“净值日期”，则从 A 列第一个有效日期开始导入。
        - 有效日期限定在 2000-01-01 到 2099-12-31 之间，避免普通数字被误判。
        - 如果两种方式都找不到起始行，则该源文件不会导入有效数据。

4. 最新产品数据表

    STEP3 会输出产品数据表：

        产品净值汇总_yyyymmdd.xlsx

    STEP4 和 STEP5 会自动查找同目录下日期最大的 产品净值汇总_yyyymmdd.xlsx 作为最新产品数据表。

    注意：

        - STEP4 依赖该文件生成图表。
        - STEP5 依赖该文件导出和拼接图表。
        - 如果同目录存在多个 产品净值汇总_yyyymmdd.xlsx，会使用文件名日期最大的那个。

5. 四个图表模板

    STEP4 必须依赖以下 4 个 .crtx 图表模板，且必须放在主 .xlsm 同目录：

        净值图表_红.crtx
        净值图表_蓝.crtx
        收益率图表_红.crtx
        收益率图表_蓝.crtx

    缺少任意一个模板，STEP4 会中止。

6. 产品图表输出文件夹

    STEP5 会根据最新产品数据表日期自动生成：

        产品图表_yyyymmdd\
        产品图表_yyyymmdd\raw\

    STEP6 会自动查找日期最大的 产品图表_yyyymmdd 文件夹作为最新图片来源。

7. 需要更新的折线图展示文件夹和 4 个展示文件

    展示模板文件必须位于主 .xlsm 同目录下：

        折线图展示\

    STEP6 会在该文件夹中按以下 4 类文件名前缀查找最新模板文件，并另存为当前图片日期的新文件：

        家族家庭-稳享长期限产品收益展示-yyyymmdd.xlsx
        交通银行-产品收益展示-yyyymmdd.xlsx
        直销-汇益系列产品收益展示-yyyymmdd.xlsx
        江苏银行-圆融安享产品收益展示-yyyymmdd.xlsx

    注意：

        - 文件名前缀必须与上面完全一致。
        - yyyymmdd 为 8 位日期。
        - 如果同一类存在多个日期版本，STEP6 会使用日期最大的模板文件。
        - STEP6 保存的新文件日期来自最新 产品图表_yyyymmdd 文件夹。

8. STEP6 指定图片依赖

    STEP6 会从最新 产品图表_yyyymmdd 文件夹读取以下图片：

        产品图表_yyyymmdd\汇益稳享364天101号_红.png
        产品图表_yyyymmdd\汇益稳享728天108号_红.png
        产品图表_yyyymmdd\汇益稳健7天2号_红.png
        产品图表_yyyymmdd\汇益稳健28天6号_红.png
        产品图表_yyyymmdd\汇益稳享91天3号_红.png
        产品图表_yyyymmdd\汇益稳健日开101号_红.png
        产品图表_yyyymmdd\汇益稳健28天101号_红.png
        产品图表_yyyymmdd\交鑫致远6个月101号_红.png
        产品图表_yyyymmdd\raw\圆融安享日开8号_净值_蓝.png
        产品图表_yyyymmdd\raw\圆融安享7天2号_净值_蓝.png
        产品图表_yyyymmdd\raw\圆融安享28天1号_净值_蓝.png

    缺少任意指定图片，STEP6 会报错中止。


二、总体运行方式
----------------

推荐运行顺序：

    STEP1 -> STEP2 -> STEP3 -> STEP4 -> STEP5 -> STEP6

控制台入口宏：

    打开控制台

历史说明：

    - 早期说明中曾使用入口名 打开调度控制台。
    - 当前已测试可用的入口名为 打开控制台。
    - 两者本质都是调用 frmDispatchConsole.Show，只需保留一个实际可用入口即可。

如果已经建立用户窗体控制台，可通过该入口打开统一调度窗口，然后选择：

    1. 执行选中步骤
    2. 按顺序执行全部步骤
    3. 单独调用三个工具按钮

注意：

    - 一键执行全部步骤只应包含 STEP1 到 STEP6。
    - 三个工具不应自动包含在主流程中，应由操作人员按需手动执行。
    - 手工单独执行某个步骤前，必须确认该步骤依赖的前置产物已经存在。


三、基础依赖
------------

1. 工作簿要求

    - 当前主工作簿应为 .xlsm 格式。
    - 必须启用宏。
    - 所有外部源文件、模板文件、输出文件默认都以当前 .xlsm 所在目录为基准。

2. 工作表要求

    - 必须存在 Sheet1。
    - 必须存在 Sheet2。
    - STEP2 会创建或清空重建 Sheet3。

3. Sheet1 维度表格式

    Sheet1 从第 2 行开始读取产品维度。

    - A 列：产品编号
    - B 列：产品名称
    - C 列：产品简称

    注意：

    - STEP2 使用 Sheet1 将产品编号映射到产品简称。
    - STEP3 按 Sheet1 中的产品顺序导出产品 sheet。
    - STEP3 要求产品编号和产品简称都不能为空。

4. Sheet2 数据主表格式

    Sheet2 使用 A:J 作为净值数据主表。

    关键列：

    - A 列：净值日期
    - B 列：产品编号
    - D 列：净值
    - G 列：30 日年化收益率

    注意：

    - STEP1 会向 Sheet2 增量合并数据。
    - TOOL1 和 TOOL3 会直接修改 Sheet2。
    - 执行会删除或改写 Sheet2 的工具前，建议先备份工作簿。

5. 外部文件和文件夹依赖

    净值源文件必须放在 .xlsm 同目录，文件名格式必须为：

        净值数据浏览表 yyyy-mm-dd至yyyy-mm-dd.xlsx

    最新产品数据表由 STEP3 输出，文件名格式为：

        产品净值汇总_yyyymmdd.xlsx

    图表模板必须放在 .xlsm 同目录：

        净值图表_红.crtx
        净值图表_蓝.crtx
        收益率图表_红.crtx
        收益率图表_蓝.crtx

    展示文件必须位于 .xlsm 同目录下的文件夹：

        折线图展示\

    折线图展示\ 中需要存在以下 4 类展示文件：

        家族家庭-稳享长期限产品收益展示-yyyymmdd.xlsx
        交通银行-产品收益展示-yyyymmdd.xlsx
        直销-汇益系列产品收益展示-yyyymmdd.xlsx
        江苏银行-圆融安享产品收益展示-yyyymmdd.xlsx

    产品图表输出文件夹由 STEP5 自动生成，格式为：

        产品图表_yyyymmdd\
        产品图表_yyyymmdd\raw\


四、六个主流程
--------------

1. STEP1导入净值数据

    模块文件：

        1.导入净值数据.bas

    功能：

        扫描当前 .xlsm 同目录下所有符合命名规则的净值源文件：

            净值数据浏览表 yyyy-mm-dd至yyyy-mm-dd.xlsx

        从每个源文件的 Sheet1 中自动识别数据起始行，读取起始行到末行的 A:J 数据，并按主键合并到当前工作簿 Sheet2。

    主键规则：

        净值日期 + 产品编号

    处理逻辑：

        - 如果 Sheet2 中已存在相同主键，则覆盖更新该行数据。
        - 如果 Sheet2 中不存在该主键，则追加新增。
        - 源文件按开始日期升序处理。
        - 源数据起始行优先通过 A 列“净值日期”表头判断。
        - 如果找不到“净值日期”，则使用 A 列第一个有效日期所在行。
        - 有效日期限定在 2000-01-01 到 2099-12-31 之间。
        - 最终按日期和产品编号排序。

    输入：

        - 同目录下的 净值数据浏览表 *.xlsx
        - 每个源文件的 Sheet1
        - 源文件自动识别出的数据区 A:J

    输出：

        - 当前工作簿 Sheet2

    注意事项：

        - 源文件名必须严格匹配命名格式。
        - 如果源文件被占用，可能无法打开并被跳过。
        - 如果源文件 Sheet1 的 A 列找不到“净值日期”，也找不到有效日期，则不会导入该源文件的数据。
        - Sheet2 第一行应作为表头保留。


2. STEP2生成数据报告

    模块文件：

        2.生成数据报告.bas

    功能：

        基于 Sheet2 中的净值数据，按产品编号汇总数据完整度和净值统计，并输出到 Sheet3。

    输入：

        - Sheet1：产品编号、产品简称
        - Sheet2：净值数据主表

    输出：

        - Sheet3：数据完整度报告

    报告字段包括：

        - 产品编号
        - 产品简称
        - 净值日期最小值
        - 净值日期最大值
        - 净值最小值
        - 净值最大值
        - 数据条数
        - 总跨度(天)
        - 最大连续缺失天数
        - 首日净值
        - 末日净值
        - 累计涨跌幅(%)

    注意事项：

        - 如果 Sheet3 不存在，会自动创建。
        - 如果 Sheet3 已存在，会清空后重建。
        - Sheet2 无数据或无有效数据时会中止。


3. STEP3导出产品数据

    模块文件：

        3.导入净值数据.bas

    公开过程：

        STEP3导出产品数据

    功能：

        根据 Sheet1 维度表中的产品列表，将 Sheet2 中的历史净值数据按产品拆分导出到一个新的 xlsx 文件。

    输出文件命名：

        产品净值汇总_yyyymmdd.xlsx

    其中 yyyymmdd 通常取 Sheet2 中最大净值日期。

    处理逻辑：

        - 按 Sheet1 中的产品顺序生成 sheet。
        - 每个产品一个 sheet。
        - 每个 sheet 以产品简称命名。
        - 产品内数据按净值日期升序排列。
        - 对重复主键做去重，后出现的数据覆盖先出现的数据。
        - 对净值列做分红日平滑。
        - 基于平滑后的净值重算 30 日年化收益率。
        - 30 日年化收益率的填补统计会写入 Sheet3 的相关列。

    输入：

        - Sheet1：产品编号、产品简称
        - Sheet2：A:J 净值数据主表

    输出：

        - 同目录下 产品净值汇总_yyyymmdd.xlsx
        - Sheet3 中可能增加 30 日年化收益率填补统计信息

    注意事项：

        - Sheet1 中没有数据会中止。
        - Sheet1 中产品编号或产品简称为空的产品不会被有效导出。
        - Sheet2 中没有数据会中止。
        - 如果同名 产品净值汇总_yyyymmdd.xlsx 已存在，会先删除再保存新文件。
        - Sheet2 中存在但 Sheet1 未配置的产品会被跳过，并在完成提示中列出。
        - Sheet1 中配置但 Sheet2 无数据的产品会被跳过，并在完成提示中列出。


4. STEP4生成产品图表

    模块文件：

        4.生成产品图表.bas

    功能：

        打开同目录下最新的 产品净值汇总_yyyymmdd.xlsx，为每个产品 sheet 生成净值和收益率图表。

    图表类型：

        - 净值图表_红
        - 净值图表_蓝
        - 收益率图表_红
        - 收益率图表_蓝

    依赖模板：

        - 净值图表_红.crtx
        - 净值图表_蓝.crtx
        - 收益率图表_红.crtx
        - 收益率图表_蓝.crtx

    输入：

        - 最新的 产品净值汇总_yyyymmdd.xlsx
        - 同目录下 4 个 .crtx 图表模板

    输出：

        - 直接保存到 产品净值汇总_yyyymmdd.xlsx 内部
        - 每个产品 sheet 上生成对应图表

    注意事项：

        - 执行前必须先完成 STEP3。
        - 如果缺少任一图表模板，会中止。
        - 如果找不到 产品净值汇总_yyyymmdd.xlsx，会中止。
        - 每次执行会删除每个产品 sheet 上已有 chart，然后重新生成。
        - 收益率图表会跳过开头连续为 0 的数据。
        - 收益率图表的 Y 轴下限可能被强制设为 0，涉及产品会在完成提示中列出。


5. STEP5导出拼接图表

    模块文件：

        5.导出拼接图表.bas

    功能：

        从最新的 产品净值汇总_yyyymmdd.xlsx 中导出每个产品 sheet 上的图表，并按颜色拼接。

    输入：

        - 最新的 产品净值汇总_yyyymmdd.xlsx
        - 该文件内每个产品 sheet 上由 STEP4 生成的图表

    输出文件夹：

        产品图表_yyyymmdd\
        产品图表_yyyymmdd\raw\

    输出内容：

        - raw 文件夹中保存单张原始图表 png。
        - 产品图表_yyyymmdd 文件夹中保存拼接后的红色图、蓝色图。

    图表命名依赖：

        - chart_净值_红
        - chart_收益率_红
        - chart_净值_蓝
        - chart_收益率_蓝

    注意事项：

        - 执行前必须先完成 STEP4。
        - 如果找不到 产品净值汇总_yyyymmdd.xlsx，会中止。
        - 如果某产品缺少收益率图，会只复制净值图作为输出。
        - 同名图片会被覆盖。
        - 代码会临时开启屏幕刷新以保证 chart 正常渲染。


6. STEP6导入全部展示图片

    模块文件：

        6.导入展示图片.bas

    功能：

        查找最新的 产品图表_yyyymmdd 文件夹，将指定产品图表插入到 折线图展示\ 下的展示文件中，并另存为当前图片日期的新文件。

    输入：

        - 最新的 产品图表_yyyymmdd\
        - 最新的 产品图表_yyyymmdd\raw\
        - 折线图展示\ 下的展示模板文件

    输出：

        - 折线图展示\ 下新的展示结果文件

    当前处理的展示文件类型：

        - 家族家庭-稳享长期限产品收益展示-yyyymmdd.xlsx
        - 交通银行-产品收益展示-yyyymmdd.xlsx
        - 直销-汇益系列产品收益展示-yyyymmdd.xlsx
        - 江苏银行-圆融安享产品收益展示-yyyymmdd.xlsx

    插入逻辑：

        - 默认处理展示工作簿的第一个工作表。
        - 删除目标区域内原有图片。
        - 插入指定 png 图片。
        - 图片按目标区域宽度等比例缩放。
        - 根据图片高度调整目标行行高。
        - 另存为带图片日期的新文件。

    注意事项：

        - 执行前必须先完成 STEP5。
        - 必须存在 产品图表_yyyymmdd 文件夹。
        - 必须存在代码中指定的图片文件。
        - 必须存在 折线图展示\ 文件夹和匹配前缀的展示模板文件。
        - 如果同名展示结果文件已存在，会先删除再保存。
        - 目标区域内旧图片会被删除。
        - 完成窗口会输出处理完成的展示文件路径。


五、三个工具
------------

1. 清洗重复数据

    模块文件：

        TOOL1清洗重复数据.bas

    功能：

        清洗 Sheet2 中的重复数据和无效数据。

    主键规则：

        净值日期 + 产品编号

    处理逻辑：

        - 倒序扫描 Sheet2。
        - 如果主键重复，保留物理位置靠后的记录。
        - 删除主键无效的数据。
        - 清洗后重新写回 Sheet2。
        - 最终按日期和产品编号升序排序。

    注意事项：

        - 该工具会直接修改 Sheet2。
        - 如果 Sheet2 数据少于 2 行，会提示无需去重。
        - 如果清洗后没有任何有效数据，会中止。
        - 建议执行前备份工作簿。


2. 输出更新对象

    模块文件：

        TOOL2输出更新对象.bas

    功能：

        用于排查最新净值源文件和 Sheet2 最近日期数据情况。

    输出位置：

        VBE 即时窗口

    查看方式：

        在 VBA 编辑器中按 Ctrl + G 打开“即时窗口”。

    输出内容包括：

        - Excel 版本
        - 最新净值源文件名
        - 文件名结束日期
        - 源文件数据末行
        - 源文件末尾 10 行 A/B 列详情
        - 源文件中各日期记录数
        - Sheet2 中最近 10 个日期的记录数

    注意事项：

        - 该工具不弹出结果窗口，主要看即时窗口。
        - 如果没有找到符合命名规则的净值源文件，会直接退出。
        - 会以只读方式打开最新净值源文件。


3. 批量删除数据

    模块文件：

        TOOL3批量删除数据.bas

    功能：

        按用户输入的产品编号，删除 Sheet2 中该产品编号的全部记录。

    操作流程：

        1. 输入产品编号。
        2. 扫描 Sheet2。
        3. 显示匹配记录条数和日期范围。
        4. 二次确认。
        5. 删除匹配行。

    注意事项：

        - 该工具会直接删除 Sheet2 行。
        - 删除操作不可撤销。
        - 二次确认窗口默认按钮为“否”。
        - 执行前必须确认产品编号准确。
        - 强烈建议执行前备份工作簿。


六、常见问题排查
----------------

1. 提示找不到净值数据浏览表

    检查：

        - 文件是否放在 .xlsm 同目录。
        - 文件名是否符合 净值数据浏览表 yyyy-mm-dd至yyyy-mm-dd.xlsx。
        - 日期格式是否为 yyyy-mm-dd。
        - 文件扩展名是否为 .xlsx。

2. STEP3 未导出某些产品

    检查：

        - Sheet1 是否配置了产品编号和产品简称。
        - Sheet2 是否存在该产品编号的数据。
        - 产品编号是否有前后空格或格式差异。

3. STEP4 提示缺少模板

    检查同目录是否存在：

        - 净值图表_红.crtx
        - 净值图表_蓝.crtx
        - 收益率图表_红.crtx
        - 收益率图表_蓝.crtx

4. STEP5 导出图片异常或空白

    检查：

        - 是否已先运行 STEP4。
        - 产品汇总文件中是否存在 chart_净值_红 等图表对象。
        - Excel 是否能正常渲染图表。

5. STEP6 找不到展示模板或图片

    检查：

        - 是否已先运行 STEP5。
        - 是否存在最新 产品图表_yyyymmdd 文件夹。
        - 指定图片文件名是否存在。
        - 折线图展示\ 文件夹是否存在。
        - 展示文件前缀是否与代码一致。


七、覆盖和删除风险汇总
----------------------

以下操作会覆盖、清空或删除数据，应特别注意：

    - STEP2 会清空并重建 Sheet3。
    - STEP3 如同名 产品净值汇总_yyyymmdd.xlsx 已存在，会先删除再保存。
    - STEP4 会删除产品汇总文件每个 sheet 上已有图表后重建。
    - STEP5 会覆盖同名导出图片。
    - STEP6 会覆盖同名展示结果文件，并删除目标区域内旧图片后插入新图片。
    - 清洗重复数据 会直接改写 Sheet2。
    - 批量删除数据 会直接删除 Sheet2 中指定产品编号的全部记录。

建议：

    - 每日正式运行前，备份主 .xlsm。
    - 执行 STEP3、STEP6 和批量删除数据前，确认目标文件或产品编号无误。
    - 出现异常时，优先查看弹窗提示和 VBE 即时窗口。


八、建议日常操作流程
--------------------

1. 将最新 净值数据浏览表 yyyy-mm-dd至yyyy-mm-dd.xlsx 放到 .xlsm 同目录。
2. 打开主 .xlsm，并启用宏。
3. 运行 打开控制台。
4. 按顺序执行 STEP1 到 STEP6。
5. 如需排查更新范围，单独运行 输出更新对象，并查看 VBE 即时窗口。
6. 如发现 Sheet2 有重复数据，先备份，再运行 清洗重复数据。
7. 如需删除某产品历史数据，先备份，再运行 批量删除数据。

十、调度控制台配置说明
----------------------

当前已采用手工创建用户窗体的方式实现调度台，并已测试可以正常运行。

1. 普通模块入口

    在普通模块中加入以下代码：

        Option Explicit

        Public Sub 打开控制台()
            frmDispatchConsole.Show
        End Sub

    说明：

        - frmDispatchConsole 是用户窗体名称。
        - 打开控制台 是日常运行控制台的入口宏。
        - 可将 打开控制台 绑定到 Excel 工作表按钮、快捷键或快速访问工具栏。

2. 用户窗体名称

    用户窗体 Name 必须设置为：

        frmDispatchConsole

    Caption 可设置为：

        净值流程控制台

3. 用户窗体控件清单

    必须存在以下控件，Name 必须完全一致：

        ListBox:
            Name = lstSteps

        CommandButton:
            Name = cmdRunSelected
            Caption = 执行选中步骤

        CommandButton:
            Name = cmdRunAll
            Caption = 按顺序执行全部步骤

        CommandButton:
            Name = cmdToolCleanDup
            Caption = 清洗重复数据

        CommandButton:
            Name = cmdToolPrintUpdate
            Caption = 输出更新对象

        CommandButton:
            Name = cmdToolDelete
            Caption = 批量删除数据

        CommandButton:
            Name = cmdClose
            Caption = 关闭

        Label:
            Name = lblStatus
            Caption = 请选择步骤后执行，或直接按顺序执行全部步骤。

    注意：

        - 控件位置和大小可以自行调整。
        - 控件 Name 必须一致，否则代码中的事件过程和状态更新会找不到控件。
        - 如果窗体中额外出现 ListBox1_Click、Label1_Click 等空事件过程，可以保留，也可以删除，不影响主流程。

4. 控制台运行逻辑

    初始化时，lstSteps 会加载 6 个主流程：

        1. STEP1 导入净值数据
        2. STEP2 生成数据报告
        3. STEP3 导出产品数据
        4. STEP4 生成产品图表
        5. STEP5 导出拼接图表
        6. STEP6 导入全部展示图片

    对应宏名：

        - STEP1导入净值数据
        - STEP2生成数据报告
        - STEP3导出产品数据
        - STEP4生成产品图表
        - STEP5导出拼接图表
        - STEP6导入全部展示图片

    按钮行为：

        - 执行选中步骤：只运行 lstSteps 当前选中的一个步骤。
        - 按顺序执行全部步骤：从 STEP1 到 STEP6 依次运行。
        - 清洗重复数据：单独运行工具 清洗重复数据。
        - 输出更新对象：单独运行工具 输出更新对象。
        - 批量删除数据：单独运行工具 批量删除数据。
        - 关闭：关闭控制台窗体。

5. Application.Run 调用方式

    控制台使用 Application.Run 根据宏名调用流程和工具。

    优点：

        - 窗体代码不需要直接引用每个过程。
        - 后续如果模块文件名变化，只要公开过程名不变，控制台仍可运行。
        - 如果宏执行失败，会在统一错误窗口中显示错误编号和错误描述。

    注意：

        - mStepMacros 中的宏名必须与公开过程名完全一致。
        - 中文宏名不能多空格、少字或错字。
        - 如果某个按钮报错找不到宏，应先检查宏名和模块是否已导入到当前工作簿。

6. 控制台状态显示

    lblStatus 用于显示当前状态，格式为：

        yyyy-mm-dd hh:nn:ss  正在执行: xxx
        yyyy-mm-dd hh:nn:ss  已完成: xxx
        yyyy-mm-dd hh:nn:ss  执行失败: xxx

    执行期间 SetBusy 会临时禁用按钮和列表，避免重复点击导致流程重入。

7. 控制台注意事项

    - 按顺序执行全部步骤前，会弹出确认窗口。
    - 三个工具不会包含在一键全部流程中，必须手动点击。
    - 批量删除数据仍会触发自身的输入框和二次确认。
    - 如果某一步失败，全部顺序执行会在失败处停止，不继续执行后续步骤。
    - 控制台只负责调度，不改变各流程自身的输入输出规则和覆盖风险。
