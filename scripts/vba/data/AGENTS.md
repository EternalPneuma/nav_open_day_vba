# scripts/vba/data 指引

## 定位
- 本目录是新版数据侧 VBA 流水线，模块是可复制到 Excel VBE 的文本源码；当前文件无 `.txt` 扩展名。
- 当前顺序：`01_auto_input` → `02_calculate_open_date` → `03_output_date` → `04_output_report`。
- 相邻的 `scripts/vba/chart/` 是图表/图片/展示输出流水线；不要把两者混成一个模块，合并项目时先保持边界清晰。

## 模块入口
- `01_auto_input`：运行 `STEP1导入增量净值数据` 或别名 `S1ImportNAV`，从 `181-固收上层` 增量导入 `净值数据`。
- `02_calculate_open_date`：运行 `STEP2测算开放日` 或别名 `S2CalculateOpenDays`，写入下一/上一/上上一开放日、实际间隔、基准日期净值等列。
- `03_output_date`：运行 `STEP3产品分类表现报告` 或别名 `S3ProductReport`，输出 `yyyyMMdd-上层产品分类表现.xlsx`。
- `04_output_report`：运行 `Step04OutputReport`，读取 `yyyyMMdd-上层产品分类表现.xlsx`，输出 `yyyyMMdd-展示.xlsx`。

## 依赖和字段
- `01_auto_input` 依赖源文件命名 `HS-181_多账套净值查询_yyyyMMdd.xlsx`、目标 sheet `净值数据`、字段 `日期`、审计列 `source_file/source_sheet`，并扫描前 30 行找表头。
- `02_calculate_open_date` 依赖 `产品分类`、`开放日`、`净值数据`，关键字段包括 `序号`、`开放日`、`信托计划代码`、`日期`、`单位净值`。
- `03_output_date` 依赖 `产品分类`、`净值数据`、`产品信息`，并读取 `理论间隔`、`基准收益率`、`成立日` 等中文字段。
- `04_output_report` 依赖 `03_output_date` 的输出文件和仓库根目录 `title.png`；展示工作簿包含 `稳享长期限`、`直销`、`交行代销` 三个 sheet。

## 04 展示报表约定
- 基准日期沿用 `产品分类` 表中的 `基准日期`，输出文件为 `yyyyMMdd-展示.xlsx`。
- 头图从 `ThisWorkbook.Path & "\title.png"` 插入，覆盖 A1:G2；图片保持比例，按最终列宽插入，`Placement` 使用只移动不随单元格缩放的设置。
- `稳享长期限` 使用 `03_output_date` 中 `稳享长期限` sheet 的数据，展示 `产品名称`、`下一开放日`、`开放频率`、`当前周期运作天数`、`当前周期年化`；`开放频率` 对应源表 `理论间隔`。
- `直销` 按源表 C 列 `系列` 拆分为 `汇益稳健系列兑付业绩`、`汇益系列兑付业绩`、`交鑫致远系列兑付业绩`，不同块的展示字段不同，按现有代码映射为准。
- `交行代销` 按源表 C 列 `系列` 拆分为 `汇益稳健系列业绩`、`汇益系列业绩`、`汇益稳享系列业绩`、`蓝色港湾系列业绩`；标题由系列名加 `系列业绩` 组成。
- `直销` 和 `交行代销` 中理论间隔为 `7` 或 `每周` 的产品，展示的 `下一开放日` 不取源表值，而是基准日期之后的下一个周三；若基准日当天是周三，则取下一周三。
- 数据区统一使用微软雅黑、金色 `#F4DC8B`、深红 `#8B1919` 和 `#70130A`；深红按开放频率分组交替使用，相同开放频率在展示列中合并居中。
- 数据来源备注行使用另一种深红背景并加白色边框；风险提示行金色、字号 10、B:F 合并左对齐。

## 编码和修改边界
- 保持中文 sheet 名、字段名、过程注释和 MsgBox 文案；不要为了“规范化”翻译或批量改名。
- 继续使用常量集中声明 sheet/字段名，方便后续数据库合并后调整映射。
- 优先做小范围、可复制到 VBE 的改动；除非用户明确要求，不要引入打包或自动导入 VBE 流程。
- 不要用自动 Excel COM/VBE 测试改写模块，避免中文编码被破坏；验证以静态检查、字段核对和用户手动 Excel 运行为主。
